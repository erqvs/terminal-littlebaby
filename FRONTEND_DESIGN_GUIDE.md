# terminal-littlebaby 前端开发手册

本文档用于约束 `terminal-littlebaby` 的前端开发方式，保证后续新增页面、重构页面、让其他 AI 协作开发时，产出的界面风格保持一致。

适用范围：

- 项目：`/path/to/project/frontend`
- 技术栈：React 19 + TypeScript + Vite + Ant Design 6
- 设计基线：Ant Design 暗色主题之上，叠加少量全局覆写和大量页面级内联样式

如果目标是重新设计、品牌升级或视觉改版，不要按本文档执行。本文档只服务于一件事：让后续开发出来的前端继续保持现在这套样式和气质。

## 1. 总体设计定位

`terminal-littlebaby` 的前端不是营销型页面，也不是强调品牌包装的 SaaS 官网。它本质上是一个偏个人工具型、管理后台型、桌面优先的暗色控制台。

设计气质必须满足下面几点：

- 第一印象是“克制、直接、功能优先”，不是“炫技、装饰性强、视觉实验”
- 整体以深色背景承载信息，高亮只给关键操作和状态反馈
- 卡片、表格、表单、图表都是标准后台组件，不做过度异形设计
- 页面重点不是品牌视觉，而是数据浏览、录入和状态管理
- 视觉风格允许少量细节变化，但不能出现大幅品牌化改造

一句话概括：

这是一个“深色财务/任务管理面板”，而不是一个“设计作品集式的网页应用”。

## 2. 开发实现原则

这个项目的真实实现方式有三个层次：

1. 全局主题用 Ant Design `ConfigProvider`
2. 应用壳层和少量基础样式放在 `App.css` / `index.css`
3. 页面细节大量依赖 JSX 内联 `style`

开发时必须保留这个倾向：

- 优先继续使用 Ant Design，而不是自己重写整套组件
- 优先使用 Ant Design 的暗色算法，而不是手搓完整主题系统
- 页面局部差异用内联样式处理，而不是引入新的 CSS-in-JS 方案
- 只把稳定的壳层规则放进全局 CSS

不要擅自改成以下方案：

- 不要改成 Tailwind 主导的视觉体系
- 不要改成亮色主题
- 不要加品牌级渐变背景、发光、玻璃拟态、插画
- 不要把所有页面重构成统一“设计系统演示模板”

## 3. 全局设计令牌

虽然项目里没有把令牌集中写成单独文件，但实际视觉令牌很稳定，可以按下面执行。

### 3.1 主题核心

- Ant Design 主题算法：`theme.darkAlgorithm`
- 主色：`#1890ff`
- 全局圆角基准：`8px`

### 3.2 背景层级

- 页面总背景：`#141414`
- 侧边栏背景：`#1f1f1f`
- 登录卡片背景：`#1f1f1f`
- 较亮的弱对比区块：`#262626`
- 浅层浮起背景常用：`rgba(255,255,255,0.04)` 到 `rgba(255,255,255,0.08)`

推荐分层理解：

- L0 页面基底：`#141414`
- L1 侧栏 / 卡片 / 面板：`#1f1f1f`
- L2 弱强调块：`#262626` 或半透明白叠层

### 3.3 边框与分割线

- 强分割线：`#303030`
- 弱分割线：`rgba(255,255,255,0.04)`
- 轻边框：`rgba(255,255,255,0.06)`、`rgba(255,255,255,0.08)`、`rgba(255,255,255,0.10)`

规则：

- 常规卡片依赖 Ant Design 自身暗色卡片风格
- 结构性边界优先用细边框，不依赖阴影堆层
- 只有登录卡片等少数区块使用明显阴影

### 3.4 文本颜色

- 主标题 / 高优先级文本：`#fff` 或 `rgba(255,255,255,0.85)`
- 普通正文：默认白色体系
- 次级说明：`rgba(255,255,255,0.65)` 或 `rgba(255,255,255,0.45)`
- 更弱说明：`#666` 或 `#888`

建议优先级：

- 一级：`#fff`
- 二级：`rgba(255,255,255,0.85)`
- 三级：`rgba(255,255,255,0.65)`
- 四级：`rgba(255,255,255,0.45)`
- 弱提示：`#666`

### 3.5 语义色

- 主操作 / 信息 / 导航高亮：`#1890ff`
- 成功 / 收入 / 正向金额：`#52c41a`
- 危险 / 支出 / 负债 / 删除：`#ff4d4f`
- 警告 / 接近阈值：`#faad14` 或 `orange`
- 组合类标签：`cyan`
- 论文标签等少量辅助：`purple` / `geekblue`

开发时必须保持语义稳定：

- 蓝色不是“成功色”，它是主交互和信息色
- 绿色明确代表正向增长、收入、剩余安全
- 红色明确代表支出、删除、风险、超额、负债

### 3.6 圆角

- 全局基础圆角：`8px`
- 登录卡片：`12px`
- 课表卡片：`12px`
- 小色块、账户头像块：`4px` 或 `8px`
- 状态胶囊：`999px`

开发约束：

- 这套 UI 不用大圆角，不用胶囊化大面积按钮，不走移动端 App 风格

### 3.7 阴影

- 大多数页面几乎不依赖阴影
- 登录页卡片会用：`0 4px 24px rgba(0, 0, 0, 0.5)`
- 课表课程卡片默认也尽量无阴影，主要靠底色和边框

开发约束：

- 边框和明度差异比阴影更重要

### 3.8 字体

当前项目使用系统字体栈：

```css
-apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, 'Helvetica Neue', Arial, sans-serif
```

开发时继续使用系统字体，不要额外引入品牌字体。

## 4. 图标风格

本项目大量导航和局部图标不是来自 Ant Design Icons，而是自定义 SVG。

图标规则：

- 线性描边图标
- `stroke="currentColor"`
- 常规尺寸以 `16x16` 为主，局部 `14x14` 或 `18x18`
- 描边宽度一般为 `2`
- 线端和拐角圆润：`strokeLinecap="round"`、`strokeLinejoin="round"`

视觉上要接近简洁的 Lucide/Feather 风格，而不是实心图标或高装饰图标。

开发要求：

- 导航图标、操作图标优先继续用线性图标
- 不要混入大批实心图标导致风格断裂
- 图标默认跟随文字颜色变化

## 5. 应用壳层布局

### 5.1 总体结构

应用使用左侧固定侧边栏 + 右侧内容区的后台结构。

布局要求：

- 整个应用最小高度 `100vh`
- 侧边栏固定在左侧，`sticky`，高度 `100vh`
- 右侧内容区沿用深色背景 `#141414`
- 内容区容器使用统一外边距和内边距

### 5.2 侧边栏

侧边栏行为：

- 支持折叠
- 展开时显示 `terminal-littlebaby`
- 折叠时显示 `TC`
- 菜单为分组树结构
- 底部固定“退出登录”按钮

样式规则：

- 侧边栏底色：`#1f1f1f`
- Logo 高度：`64px`
- Logo 背景：`rgba(255,255,255,0.08)`
- Logo 底边：`1px solid rgba(255,255,255,0.08)`
- 底部退出区顶部分割线：`1px solid #303030`

导航信息结构：

- 仪表盘
- 任务
- 课程
- 财务

财务下再分：

- 账户
- 负债
- 分类
- 预算
- 目标
- 历史记录

开发时不要改成：

- 顶部导航
- 左右双栏
- 移动端底部 Tab
- 卡片式首页入口导航

### 5.3 内容区

内容区统一样式：

- 外边距：`24px`
- 内边距：`24px`
- 背景仍为 `#141414`
- 圆角：`8px`

这意味着：

- 内容不是白底卡片容器套整页
- 整个页面直接坐在深色背景之上
- 只有局部模块才是 Card

## 6. 页面头部模板

几乎所有页面都遵循同一套页头模式。

标准格式：

- 左侧是页面标题 `h2`
- 如有必要，标题下再加一行浅色说明文字
- 右侧放主要操作按钮、筛选器或切换器
- 容器使用 `display: flex`
- 常见间距 `marginBottom: 24`
- 当右侧控件较多时允许 `gap: 16` 和 `flexWrap: 'wrap'`

标题风格：

- `h2`
- `margin: 0`
- 颜色通过全局样式继承为 `rgba(255,255,255,0.85)`
- 字重 600

这是整套应用里最稳定的骨架之一，必须保留。

## 7. 登录页设计

登录页是唯一一个脱离主壳层的独立页面。

### 7.1 布局

- 整页垂直水平居中
- 页面背景纯深色：`#141414`
- 登录面板宽度 `360px`
- 内边距 `40px`
- 圆角 `12px`
- 细边框 `1px solid rgba(255,255,255,0.06)`
- 阴影明显，但只出现于此处

### 7.2 头部视觉

- 顶部是一个 `64x64` 的图标容器
- 图标容器背景：`#262626`
- 圆角：`12px`
- 内部放绿色 `MoneyIcon`
- 主标题：`terminal-littlebaby`
- 副标题：`请输入密码登录`

### 7.3 表单

- 仅一项密码输入
- 输入框深色背景：`#141414`
- 边框：`1px solid #424242`
- 圆角：`8px`
- 主按钮高度：`44px`
- 主按钮圆角：`8px`

开发时禁止：

- 加插画
- 加渐变背景
- 做成大面积玻璃卡片
- 做成品牌官网式 Hero 登录

## 8. 卡片系统

### 8.1 卡片的角色

项目里 Card 有三类用途：

- 统计卡片
- 实体列表卡片
- 承载表格 / 图表 / 空状态的容器卡片

### 8.2 统计卡片

典型页面：

- 仪表盘
- 账户
- 负债
- 预算
- 简报历史
- 定时任务

特征：

- 使用 Ant Design `Card` + `Statistic`
- 也会有手写数值块
- 数值一般较大，重点靠颜色传达状态
- 不叠装饰，不加复杂背景

### 8.3 实体卡片

典型页面：

- 账户
- 负债
- 目标
- 预算
- 分类

共同特征：

- 常用 `hoverable`
- 网格排布，用 `Row/Col`
- 卡片内部第一行几乎总是“左侧实体信息 + 右侧操作”
- 左侧通常有一个 `40x40` 彩色方块作为头像/首字母容器
- 主要信息在左，编辑/删除放右上角
- 中间区域放进度条、金额、标签、备注等

头像方块样式统一建议：

- 尺寸：`40x40`
- 圆角：`8px`
- 文本颜色：`#fff`
- 字重：`bold`
- 字号：`16px`

### 8.4 空状态卡片

简单页面常用写法：

- 一个普通 `Card`
- `textAlign: 'center'`
- `padding: 40`
- 文本颜色 `#666`

复杂页面则直接使用 Ant Design `Empty`。

## 9. 表格系统

### 9.1 表格定位

表格页面主要是信息管理页，不追求强视觉表现，强调可读性和操作直接性。

典型页面：

- 历史记录
- 简报历史
- 定时任务
- 仪表盘最近交易

规则：

- 表格外面套 `Card`
- 删除等危险操作用红色链接按钮或红字
- 金额列用红绿区分正负
- 表格宽度充分时不做复杂卡片化改造

### 9.2 常见列风格

- 日期列固定宽度，格式 `YYYY-MM-DD`
- 类型列用颜色直接表达，不用复杂徽章
- 金额列加粗，并用 `+¥` / `-¥`
- 操作列宽度较窄，按钮轻量

### 9.3 筛选区

筛选器通常放在表格上方，单独用一个 `Card` 承载，内部使用：

- `Segmented`
- `Input.Search`
- `Space`

这种组合是这套 UI 的标准后台筛选区写法。

## 10. 表单、弹窗、抽屉

### 10.1 弹窗原则

项目大多数新增 / 编辑操作都用 `Modal + Form`。

规则：

- 默认 `footer={null}`，自己在表单底部放按钮区
- 表单布局统一 `layout="vertical"`
- 按钮区右对齐
- 常用组合：取消 + 保存
- 危险型实体在保存按钮上体现 danger 状态，比如负债页

### 10.2 字段风格

- 金额优先 `InputNumber`
- 日期优先 `DatePicker`
- 分类、账户优先 `Select`
- 颜色使用 `ColorPicker`
- 多选关系直接用多选 `Select`

### 10.3 说明文案块

当某个表单逻辑较特殊时，会插入一块浅色说明区域，而不是 tooltip。

典型表现：

- 背景：`rgba(24,144,255,0.08)` 或其他弱色块
- 圆角：`8px`
- 内边距：`12px`
- 字号：`13px`
- 文字颜色：`rgba(255,255,255,0.65)`

### 10.4 抽屉

只有辅助信息量较大的从属信息才用 `Drawer`，例如定时任务的运行记录。

不要滥用：

- 主编辑流程仍应优先使用 `Modal`

## 11. 交互控件的使用偏好

整套前端对 Ant Design 控件的偏好非常明确：

- `Button`：主操作、次操作
- `Card`：模块承载
- `Row/Col`：响应式网格
- `Statistic`：概览数据
- `Progress`：预算、课程、负债、目标
- `Tag`：分类状态、阈值、时间提示
- `Tabs` / `Segmented`：轻量切换
- `Table`：管理列表
- `Select`：筛选和录入
- `Empty`：空状态
- `FloatButton`：快速记账入口

不要换成其他重型 UI 模式，例如：

- 自定义大块筛选器
- 复杂手写下拉
- 自定义数据表格框架

## 12. 页面类型模板

这是最重要的一节。新增页面时，优先直接按这些模板落地，不要重新发明一套页面骨架。

### 12.1 登录页模板

结构：

- 全屏居中容器
- 中央单卡片
- 顶部图标块
- 标题、副标题
- 单字段表单
- 底部主按钮

### 12.2 主仪表盘模板

结构顺序：

1. 页头标题
2. 4 个核心统计卡片
3. 一张趋势图卡片
4. 两个并排概览卡片
5. 一张最近记录表格卡片

设计重点：

- 信息密度高，但仍保持后台式秩序
- 统计卡片不做大面积视觉修饰
- 图表使用红绿双线面积图

### 12.3 实体管理卡片页模板

适用：

- 账户
- 负债
- 目标
- 预算
- 分类

结构顺序：

1. 页头标题 + 新增按钮
2. 可选的概览统计卡片
3. `Row gutter={[16,16]}` 卡片网格
4. 空状态卡片或 `Empty`
5. 新增/编辑弹窗

### 12.4 表格管理页模板

适用：

- 历史记录
- 简报历史
- 定时任务

结构顺序：

1. 页头标题 + 主操作
2. 可选的标签页 / 分段控制器 / 筛选 Card
3. 可选的概览统计卡片行
4. 主表格 Card
5. 从属弹窗 / 抽屉

### 12.5 复杂网格页模板

适用：

- 课表

结构顺序：

1. 页头标题 + 周次选择 + 当前周按钮 + 状态胶囊
2. 大型 Card
3. Card 内部横向可滚动表格
4. 表格单元放课程色块卡片
5. 点击课程打开弹窗编辑

这类页面允许比普通页面更有一点视觉层次，但仍不能偏离整体克制风格。

## 13. 课表页的特殊设计规范

课表页是最特殊的页面，单独定义。

### 13.1 外层容器

- 外层仍然是普通页面结构
- 主 Card 用轻微深色渐变：
  `linear-gradient(180deg, rgba(31,31,31,0.98), rgba(20,20,20,0.96))`
- 边框：`1px solid #303030`

### 13.2 表格

- 使用真正的 `table`
- `borderCollapse: 'separate'`
- `borderSpacing: 6`
- 最小宽度：`980px`
- 允许横向滚动

### 13.3 节次列

- 左侧第一列固定为时间列
- 时间格有更明显的弱高亮背景
- 文案为“第X节 + 时间段”

### 13.4 空单元

- 使用虚线边框
- 背景极浅
- 居中显示加号和“添加课程”
- 视觉上明显比已有课程卡片更轻

### 13.5 课程卡片

激活周课程：

- 背景为课程色的低透明版本
- 边框为课程色更高透明版本
- 左侧有一条 3px 色条
- 文本颜色接近白色

非当前周课程：

- 背景更暗更淡
- 边框保留课程色但透明度更低
- 透明度略降

尺寸规则：

- 单节高度基准：`72px`
- 连续多节课程按节数累加高度

### 13.6 课表页允许的设计强化

仅限以下内容：

- 胶囊状态提示
- 轻微渐变
- 课程色块层次

仍然禁止：

- 大面积玻璃效果
- 发光阴影
- 复杂动画

## 14. 图表规范

趋势图使用 ECharts，风格要求如下：

- 深色背景下显示
- 收入线绿色，支出线红色
- 双 Y 轴
- 折线平滑
- 点为圆点
- 下方有透明渐变面积
- tooltip 使用黑底白字
- 图例文字为灰色

具体颜色：

- 收入：`#52c41a`
- 支出：`#ff4d4f`
- 轴线/网格线：`#333` / `#222`
- tooltip 背景：`rgba(0, 0, 0, 0.8)`

这张图的目标不是炫酷，而是“在暗色后台里稳定易读”。

## 15. 响应式规范

这个前端是桌面优先，但并不是完全不响应。

### 15.1 栅格断点习惯

常见写法：

- `xs={24}`
- `sm={12}`
- `md={8}`
- `lg={6}`
- `xl={8}` 或 `xl={12}`

原则：

- 手机上卡片纵向堆叠
- 平板开始两列
- 桌面三列到四列

### 15.2 页头换行

页头右侧控件多时：

- 使用 `gap`
- 使用 `flexWrap: 'wrap'`
- 不强行单行塞满

### 15.3 大表格处理

- 常规数据表格交给 Ant Design 自己处理
- 像课表这种自定义宽表必须放进 `overflowX: 'auto'` 容器

### 15.4 不要额外设计移动端导航

为了保持当前前端的一致性，不要新增：

- 底部导航栏
- 全屏抽屉菜单
- 卡片化首页总入口

当前项目主要是桌面后台风格，移动端只是能用，不是重新设计过的移动应用。

## 16. 文案与语气规范

文案风格要像一个个人管理工具：

- 直接
- 功能导向
- 少废话
- 中文简体

典型文案特征：

- 页面标题简短：`预算`、`分类`、`目标追踪`
- 按钮直接：`添加预算`、`记一笔`、`新建任务`
- 说明文字是功能说明，不是营销文案
- 空状态短而明确：`暂无账户，点击右上角添加`

禁止：

- 情绪化 slogan
- 夸张营销文案
- 大段欢迎语

## 17. 视觉一致性硬约束

如果要让其他 AI 做出几乎一样的前端，下面这些必须明确写死。

### 必须遵守

- 使用 Ant Design 暗色主题
- 主色固定为 `#1890ff`
- 页面主背景固定为 `#141414`
- 侧边栏背景固定为 `#1f1f1f`
- 以 Card、Table、Modal、Form 为主要 UI 结构
- 页面头部统一为“标题在左，操作在右”
- 财务正负语义统一使用绿和红
- 使用大量 8px / 12px 圆角，不走超大圆角
- 图标采用线性 SVG 风格
- 页面采用桌面后台式左侧导航结构

### 明确禁止

- 不要亮色主题
- 不要 Hero 大横幅
- 不要营销官网风
- 不要玻璃拟态
- 不要高饱和渐变铺满页面
- 不要改成卡片式首页导航
- 不要引入新的品牌字体
- 不要把页面改造成移动端原生 App 视觉

## 18. 交给其他 AI 的执行说明

把下面这段说明直接给其他 AI，通常就足够让它按同一风格产出。

```text
请严格遵守 terminal-littlebaby 的现有前端开发手册，不要做风格升级或二次创作。

必须满足：

- 常用组件：Card、Row/Col、Statistic、Table、Tag、Progress、Modal、Form、Segmented、Empty、Drawer
```

## 19. 后续开发边界

如果之后要继续加页面、加模块、做局部重构，建议遵循以下边界：

- 可以继续新增页面，但必须沿用当前页头模板和 Card/Table/Form 结构
- 可以把当前散落的颜色和间距逐步抽成 token，但抽完后视觉不能变化
- 可以提升代码复用度，但不要顺手改造视觉
- 可以优化移动端可用性，但不能改变整体后台风格

## 20. 提交前自检清单

在提交任何前端改动前，至少检查下面这些问题：

- 页面整体是否仍然是深色后台风格，而不是亮色页或官网页
- 页面主背景是否仍然是 `#141414` 体系，侧边栏是否仍然是 `#1f1f1f`
- 语义色是否仍然正确：蓝色交互、绿色正向、红色危险、橙色预警
- 页头是否仍然是“左标题右操作”的标准结构
- 是否优先复用了 `Card`、`Table`、`Form`、`Modal`、`Tag`、`Progress` 等现有组件模式
- 是否出现了不必要的新视觉语言，例如玻璃拟态、全屏渐变、大阴影、品牌插画
- 是否引入了不一致的圆角、按钮尺寸、图标风格或字体
- 新增页面是否能归入本文档定义的页面模板之一
- 移动端是否只是正常适配，而不是被重新设计成另一套界面
- 文案是否仍然直接、简短、工具化，没有营销语气

如果以上任意一项答案是否定的，就说明这次改动已经开始偏离 terminal-littlebaby 的前端风格，需要先收回来再继续开发。
